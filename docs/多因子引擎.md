# 多因子策略引擎使用说明

## 模块结构

```
multi_factor/engine/
├── strategy_config.py   # 策略参数（因子、池、调仓、费用）
├── data_hub.py          # Blader 数据加载
├── filters.py           # ST/停牌/新股/涨跌停过滤
├── neutralize.py        # 行业/市值中性化
├── composite.py         # 多因子加权合成
├── rebalance.py         # 调仓日程
├── backtest.py          # T+1 回测、买卖费率、滑点
├── factor_analyzer.py   # 单因子 IC / 分层
├── performance.py       # 绩效指标
├── visualization.py     # 图表
├── report.py            # CSV / JSON / HTML
└── pipeline.py          # 一键流水线
```

## HTTP API（FastAPI）

启动后访问 **Swagger**：`http://127.0.0.1:8000/docs`（标签「多因子引擎」）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/engine/backtest/run-sync` | 同步执行完整流水线 |
| POST | `/api/v1/engine/jobs/backtest` | 异步任务 |
| GET | `/api/v1/engine/backtest/performance` | 绩效指标 |
| GET | `/api/v1/engine/backtest/portfolio` | 净值 |
| GET | `/api/v1/engine/charts/{path}` | 下载 PNG |

详见 [API接口文档.md](./API接口文档.md) §6.12。

## 快速运行

```bash
python run_api.py

# 默认四因子等权、每日调仓、Top30、T+1、全A
python run_backtest.py --source ifind --engine --start 20251110 --end 20260530

# 自定义因子与持仓
python run_backtest.py --source ifind --engine \
  --factors PE_TTM,PB,ROE_TTM,MOMENTUM_20 \
  --top-n 50 --rebalance weekly --universe all_a

# YAML 配置
python run_backtest.py --source ifind --engine --strategy-config config/strategy.example.yaml
```

## 参数说明

- **手续费**：买入 0.03%、卖出 0.13%（`StrategyConfig.buy_commission` / `sell_commission`）
- **滑点**：0.1%（`slippage`）
- **T+1**：信号日收盘生成目标，下一交易日收盘成交
- **股票池**：`csi300` / `csi500` / `csi1000` 需库内 `index_members` 表；否则退化为全市场有行情标的
- **市值中性**：`--cap-neutral`，使用 `MARKET_VALUE` 因子
- **行业中性**：配置 `industry_neutral: true` 且库内有行业因子代码


沪深300/500/1000：需在库中配置 index_members 表；当前 Blader 若无成分表，csi300 等会退化为「有行情的全市场」，见 data_hub.py。
ST：无 ST 标记字段时无法从代码可靠识别；可后续在 daily 表增加 is_st 列并在 filters.py 中接入。
行业中性：需库内有行业因子（默认 SW_INDUSTRY），否则自动跳过。
旧路径 run_backtest.py --source ifind（无 --engine）仍走原 pipeline，互不影响。
